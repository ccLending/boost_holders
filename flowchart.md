# boost_holders.js 流程图

## 整体架构

该脚本的核心目标是：通过助记词批量派生子钱包，从主钱包向子钱包分发 SOL，然后用子钱包批量买入指定代币，从而增加代币的持有人数量（Holder 数）。

---

## 主流程 (`run()`)

```
┌─────────────────────────────────────┐
│              程序启动                │
│           加载 .env 配置            │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│      初始化 RPC Connection          │
│      解码主钱包私钥 (bs58)          │
│      加载主钱包 Keypair             │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   读取 checkpoint.json 获取断点序号  │
│   (支持中断后断点续跑)              │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  for i = currentIndex;              │
│      i < totalTargetCount(5000);    │
│      i += BATCH_SIZE(20)            │
│  ┌─────────────────────────────┐    │
│  │  ① 生成子钱包 (20个一批)    │    │
│  │  ② 主钱包向子钱包分发 SOL   │    │
│  │  ③ 子钱包批量买入代币       │    │
│  │  ④ 检查 Ctrl-C 退出信号    │    │
│  │  ⑤ 休眠 10 秒              │    │
│  └─────────────────────────────┘    │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│         所有批次处理完毕             │
│            程序结束                  │
└─────────────────────────────────────┘
```

## 每批次详细流程

```
┌──────────────────────────────────────────┐
│  ① generateChildWallets()                │
│  从助记词派生 20 个子钱包                 │
│  路径: m/44'/501'/{index}'/0'            │
│  输出: Keypair[] + 地址 + 私钥           │
└─────────────────┬────────────────────────┘
                  │
                  ▼
┌──────────────────────────────────────────┐
│  ② distributeRandomAmounts()             │
│  为每个子钱包生成随机 SOL 金额            │
│  范围: [minAmount, maxAmount]            │
│       (默认 0.03 ~ 0.05 SOL)            │
│  精度: 4 位小数                          │
│                                          │
│  构建一笔批量 Transfer 交易              │
│  主钱包 → 20 个子钱包                    │
│  发送并确认交易                          │
└─────────────────┬────────────────────────┘
                  │
            ┌─────┴─────┐
            │           │
         成功         失败
            │           │
            ▼           ▼
            │     ┌─────────────────┐
            │     │ 打印余额不足提示 │
            │     │ process.exit(0) │
            │     └─────────────────┘
            ▼
┌──────────────────────────────────────────┐
│  ③ executeBatchBuy()                     │
│  遍历本批 20 个子钱包，逐个执行买入       │
│                                          │
│  for each wallet:                        │
│  ┌────────────────────────────────────┐  │
│  │  查询子钱包余额                    │  │
│  │         │                          │  │
│  │    余额 <= 0.006 SOL?             │  │
│  │    ┌────┴────┐                     │  │
│  │   是         否                    │  │
│  │    │          │                    │  │
│  │  跳过    executeBuy()             │  │
│  │           │                       │  │
│  │     ┌─────┴─────┐                 │  │
│  │   成功         失败                │  │
│  │     │           │                 │  │
│  │   打印hash   recordFailedIndex() │  │
│  │              记录到 checkpoint    │  │
│  │                                   │  │
│  │  等待 2 秒 → 处理下一个钱包       │  │
│  └────────────────────────────────────┘  │
│                                          │
│  saveCheckpoint(最后序号 + 1)            │
└─────────────────┬────────────────────────┘
                  │
                  ▼
┌──────────────────────────────────────────┐
│  ④ 检查 isPendingExit                   │
│                                          │
│  用户按过 Ctrl-C?                        │
│  ┌────┴────┐                             │
│ 是         否                            │
│  │          │                            │
│  ▼          ▼                            │
│ 安全退出   ⑤ 休眠 10 秒                 │
│            然后继续下一批                 │
└──────────────────────────────────────────┘
```

## 单笔买入流程 (`executeBuy()`)

```
┌───────────────────────────────────────┐
│  计算买入金额                         │
│  buyAmount = 余额 - 0.006 SOL (Gas)  │
└─────────────────┬─────────────────────┘
                  │
                  ▼
┌───────────────────────────────────────┐
│  sdk.simulator.simulateBuy()          │
│  模拟买入，获取预期代币数量            │
└─────────────────┬─────────────────────┘
                  │
            ┌─────┴─────┐
         成功          失败 → 抛出异常
            │
            ▼
┌───────────────────────────────────────┐
│  idealTokenAmount < 1 token?          │
│  是 → 抛出异常 (买入量不足一个代币)    │
│  否 → 继续                            │
└─────────────────┬─────────────────────┘
                  │
                  ▼
┌───────────────────────────────────────┐
│  计算 maxSolAmount                    │
│  = buyAmount * 1.01 (1% 滑点容差)    │
└─────────────────┬─────────────────────┘
                  │
                  ▼
┌───────────────────────────────────────┐
│  sdk.trading.buy()                    │
│  构建买入交易                         │
└─────────────────┬─────────────────────┘
                  │
                  ▼
┌───────────────────────────────────────┐
│  设置 feePayer / recentBlockhash     │
│  签名交易                            │
│  发送 RawTransaction                 │
│  确认交易                            │
└───────────────────────────────────────┘
```

## Ctrl-C 安全退出机制

```
         用户按下 Ctrl-C (SIGINT)
                  │
            ┌─────┴──────┐
            │            │
       isSleeping     !isSleeping
       (批间休眠中)   (交易执行中)
            │            │
            ▼            ▼
      直接 exit(0)   isPendingExit = true
       安全退出      等待当前批次完成后退出
```

---

## checkpoint.json 结构

```json
{
  "currentIndex": 40,
  "failedIndices": [23, 37]
}
```

- **currentIndex**: 下次启动时从该序号继续
- **failedIndices**: 买入失败的子钱包序号列表，供后续重试

## 配置项 (`.env`)

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| NETWORK | 网络环境 | LOCALNET |
| RPC | RPC 节点地址 | http://47.109.157.92:8899 |
| MINT_ADDRESS | 目标代币地址 | - |
| SUBWALLET_COUNT_FOR_HOLDER | 目标子钱包总数 | 5000 |
| MIN_AMOUNT_FOR_HOLDER | 每个子钱包最小 SOL | 0.03 |
| MAX_AMOUNT_FOR_HOLDER | 每个子钱包最大 SOL | 0.05 |
| MNEMONIC_FOR_HOLDER | 助记词 | - |
| MAIN_WALLET_PRIVATE_KEY_FOR_HOLDER | 主钱包私钥 (bs58) | - |
